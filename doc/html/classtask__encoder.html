This run method is called by the RTOS and contains a loop in which the task checks for data and sends it if appropriate.

This method is called once by the RTOS scheduler. Each time around the for (;;) loop, it sends the current number of errors and the current count to the screen. The main point is to wait for interputs to be called in encoder_driver, which will change |error| and |count| based on different functionally.
